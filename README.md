# Прогнозирование погоды по префиксным суммам
Выполнили: Ямгуров Дмитрий, Кротевич Кира
 Анализ температуры в Амстердаме за 2025 год: данные взяты из CSV-файла за каждый день года(изначальный CSV был с температурой многих городов - выбрали Амстердам)
 
# Структура
main_weather         — главный файл, запускает всё
csv_read       — чтение CSV
pref_stat   — префиксные суммы, средние по месяцам
linear_search    — линейный поиск, сортировка месяцев
binary          — бинарное дерево поиска по температуре
stack       — скользящее среднее, стек отмены
amsterdam_2025.csv  — данные (дата, температура)

# Запуск
 - python3 weather.py
 Никаких сторонних библиотек не нужно, только встроенные
 
# Что делает программа
1. Средняя температура по месяцам — префиксные суммы
pref_stat 
make_prefix_sums строит массив накопленных сумм: prefix[i] - это сумма всех температур до дня i. Тогда сумма температур за любой отрезок дней a..b-1 находится как prefix[b] - prefix[a], без повторного перебора дней.
days_in_each_month считает, сколько дней попадает в каждый месяц (разбирает дату вида 2025-01-03 по -).
average_temp_by_month использует обе функции: зная границы месяца, сумма температур за месяц получается за одну операцию через префиксные суммы, а средняя — делением на количество дней.

2. Самый тёплый и самый холодный день — линейный поиск
linear_search, функция find_hottest_and_coldest
Один проход по списку температур: на каждом шаге сравниваем текущее значение с текущим максимумом и минимумом и при необходимости обновляем их.

3. Сортировка месяцев по средней температуре
linear_search, функция sort_months_by_temp
Сортировка вставками: 12 пар (месяц, средняя температура) сортируются по температуре по возрастанию — каждый элемент вставляется на своё место в уже отсортированной части списка.

4. Поиск дней с температурой выше порога — бинарное дерево поиска
binary 
TreeNode — узел дерева: temperature (ключ), dates (список дат с такой температурой), left/right (поддеревья).
bst_insert — рекурсивная вставка: меньшие температуры идут влево, большие — вправо, равные дописываются в список дат узла.
build_bst — строит дерево по всем 365 дням года.
collect_dates_above — рекурсивный обход дерева, собирает в список все даты с температурой выше заданного порога.
find_days_above — запускает обход и возвращает результат.
Порог по умолчанию задан в weather.py (threshold = 20.0), можно изменить.

5. Сглаживание скользящим средним (окно 7 дней)
stack  , функция moving_average
Используется deque как очередь: на каждом шаге добавляем новую температуру, и если в очереди стало больше 7 элементов — удаляем самый старый (popleft). Среднее считается по тому, что осталось в очереди.

6. Отмена сглаживания — стек
stack  , класс TransformStack
Перед сглаживанием вызывается save, который кладёт копию списка температур на стек. undo снимает эту копию обратно — так можно вернуться к данным до преобразования. Если стек пуст, undo сообщает об этом и возвращает None.
